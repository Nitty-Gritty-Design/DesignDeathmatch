# Framework Improvement Plan: V3 Overhaul

This plan covers cleanup, automation improvements, showcase redesign, and the implementation of a new validation and voting system.

## 1. Repository Cleanup & Structure
- **Target:** Root directory and `framework/` folder.
- **Issue:** Redundant files (e.g., `BRIEF.md` exists in both root and `framework/`).
- **Action:** 
  - Compare the duplicate files (BRIEF, DESIGN, RULES, TASKS).
  - Define a single source of truth (likely the root files, as `setup_run.bat` copies them directly from root).
  - Delete the redundant copies in the `framework/` folder and leave a `README.md` there explaining the architecture.
- **Target:** `.gitignore`
- **Action:** Update `.gitignore` to reflect the `docs/` pathing instead of the obsolete `showcase/` pathing (e.g., ignore `docs/assets/screenshots` but keep `.gitkeep`).

## 2. Automation & Workflow Scripts
- **Target:** `copy_runs.bat`, `sync_showcase.bat`, new script `upload_benchmark.bat`
- **Action:**
  - Update `sync_showcase.bat` to fix the `showcase/` vs `docs/` pathing bug.
  - Refine `copy_runs.bat` to map `F:\_software\_AI\DesignDeathmatch_Runs` into `docs/runs`.
  - Create a new PowerShell or Batch script (`upload_github.bat`) that detects new folders in `docs/runs` (like the new `deepseek-v4-pro` run) and automatically stages, commits, and pushes them to the repository.

## 3. Automated Result Validation Tool
- **Target:** New script `validate_run.js` (Node.js) or `validate_run.ps1`
- **Action:**
  - Build a script that scans a specific LLM run folder (e.g., `VEKTRA/v2`).
  - **Checks:** Verify all required files exist (from `TASKS.md`), check for broken links/images, verify `style.css` has no hardcoded hex colors, check if the logo animated HTML works.
  - **Output:** Generate a `validation-report.json` or HTML snippet. Deduct points for failures.
  - Integrate this report into the showcase detailed result page.

## 4. Showcase UI Overhaul (`docs/`)
- **Target:** `docs/index.html`, `docs/preview.html`, `docs/css/style.css`, `docs/js/main.js`
- **Action - Aesthetics:** 
  - Completely redesign the CSS to be neutral and grayscale. Remove any opinionated branding so the LLM designs stand out.
- **Action - Gallery (Index):**
  - **Iframes:** Replace screenshot thumbnails with live `<iframe>` embeds scaled down using CSS transforms (or simply load the V1 `index.html`).
  - **V1 Focus:** Display the V1 output in the main gallery instead of V2.
  - **Filtering & Sorting:** Add dropdowns/buttons for sorting by score/name and filtering by model family.
- **Action - Detailed View (Preview Page):**
  - Ensure V1 and V2 are both accessible, with clear explanatory text defining what they mean (e.g., "V1: Initial Run", "V2: Iterated Refinement").
  - Add missing elements generated by LLMs: links/iframes for the generative background, intro sequence, animated logo, and style guide.
  - **Pagination:** Add "Previous Result" and "Next Result" buttons to quickly flip between LLM runs.
  - Display the automated validation report.

## 5. GitHub-Backed Voting Mechanism
- **Target:** `docs/preview.html`, `docs/js/main.js`
- **Architecture:** Use the GitHub API (specifically GitHub Issues or Discussions).
- **Action:**
  - Create a UI for rating categories: Overall, Performance, Brand Coherence, etc.
  - When a user votes, the JavaScript uses the GitHub API to post a comment or react to an issue specifically created for that LLM model (e.g., an issue titled "Voting: Claude 3.7").
  - Fetch the aggregated reactions/comments from the GitHub API to display the average score on the page.
