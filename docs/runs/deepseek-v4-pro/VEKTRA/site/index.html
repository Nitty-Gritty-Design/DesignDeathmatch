What it is

One graph. Three dimensions.

Most creative tools force you to choose: are you working in sound, in image, or in logic? They give you a timeline for audio, a canvas for visuals, a code editor for behavior — and then ask you to keep them synchronized by hand.

VEKTRA refuses that separation. In VEKTRA, everything is a node in a shared signal graph. A frequency can drive a vertex position. A camera gesture can rewrite a patch parameter. Code is not a layer on top — it is the medium itself, live and inspectable at every point in the graph.

This is not a DAW with a visualizer bolted on. This is not a creative suite with an API. This is a single environment built from first principles around the idea that time, signal, and form are the same thing viewed through different lenses.

How it works

Three concepts

Signal Graph

Every element — oscillator, shader, input, output — is a node. Connections are patch cords you can inspect, reroute, and modulate. The graph is the application.

Live Time

Time is not a playback head — it is a signal you can route, multiply, divide, reverse, and synchronize across domains. Tempo is just another frequency.

Fork Everything

Every patch, every parameter, every node connection can be branched, versioned, and merged. VEKTRA's graph inherits from git: inspectable history, sharable state, infinite undo.
