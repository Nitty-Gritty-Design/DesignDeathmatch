What it is

A single environment
for signal, form, and logic

Most creative tools enforce a division. One program for sound. Another for visuals. A third for code. You spend half your time translating between them — converting a frequency into a keyframe, a shader parameter into a MIDI message. The medium dictates the method.

VEKTRA refuses. In VEKTRA, everything is a node in a shared signal graph. A 440Hz sine wave can be the carrier that drives a vertex shader. A camera's motion field can modulate a filter cutoff. Code is not a layer on top of the instrument — code is the instrument, live at every node, inspectable at every connection.

This is not a DAW with a visualization module. It is not a creative suite with an API. It is an environment built from first principles around the insight that time, signal, and form are the same phenomenon — observed through different instruments.

How it works

Three concepts that define the environment

01

The Signal Graph

Every element — oscillators, shaders, sensors, outputs — is a node. Connections are patch cords you can inspect, reroute, multiply, and modulate in real time. The graph is the application. There is no separate editor, no hidden state. What you see is what runs.

02

Time as Signal

Time is not a playback head scrolling left to right. It is a signal — routable, divisible, reversible, synchronizable. Tempo is just another frequency. A clock divider is just a node. You can modulate time the way you modulate pitch. You can drive time from audio, from video, from32 gesture.

03

Fork Everything

Every patch, every parameter, every connection inherits from git. Branch a state, explore a variation, merge what works. The entire history of a performance is a tree you can walk. Share patches as URLs. No save-version-number naming. No lost experiments. Everything is recoverable.
