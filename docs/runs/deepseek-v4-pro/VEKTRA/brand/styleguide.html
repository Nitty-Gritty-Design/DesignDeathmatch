06 Voice

VEKTRA speaks like an instrument manual written by someone who genuinely loves the instrument. The voice is technical when it needs to be — precise about milliseconds, specific about parameters — but it never confuses jargon with depth. It treats the reader as a peer, not a customer.

07 Map & Territory

This page is simultaneously documentation and demonstration. Every element on this page — from the color swatches to the type samples to the component examples — is rendered using the VEKTRA design system described on this page. The tokens you see in the code boxes are the tokens that render the boxes. The voice you read in the voice section is the voice that writes the voice section.

A designer implementing VEKTRA should be able to copy any CSS variable name from this guide, paste it into their stylesheet, and get exactly the behavior shown here.
