A terminal that someone fell in love with.

Most creative tools separate the visual from the sonic from the logical. VEKTRA treats them as one graph. A change in frequency can drive a visual parameter. A gesture can rewrite a patch. Time is explicit, not hidden. Everything is inspectable, forkable, and live.

// A VEKTRA patch is a living graph

patch.bind(lfo.output, filter.cutoff)
patch.run()

How it works

Three core concepts that define the VEKTRA environment.

01 / Graph

Graph-based patching

Every parameter is a node. Every connection is a signal. Build instruments by connecting oscillators, filters, visual planes, and logic gates in a single unified graph.

02 / Engine

Real-time generative engine

No render queue. No export dialog. What you hear is what you see. The engine runs at 60fps and audio sample rate simultaneously, with sample-accurate parameter scheduling.

03 / Material

Code as material

Write JavaScript or GLSL directly inside the patch. Code is not a plugin — it is a first-class node. Edit live, hot-reload, and fork any patch you find.

Who uses it

Built for people who make things that behave.

Electronic composer

For artists who want their instruments to surprise them. Build generative systems that evolve over hours, then perform them live.

Installation artist

For works that run for weeks unattended. VEKTRA patches can self-modify, respond to sensors, and recover from failure.

Systems developer

For the developer with a Eurorack rack and a Haskell side project. Every parameter is inspectable. Every state is serializable. Everything is forkable.
