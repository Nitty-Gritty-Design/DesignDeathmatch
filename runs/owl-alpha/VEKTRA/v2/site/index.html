01

What it is

VEKTRA is not a DAW. It is not a creative suite. It is not a plugin. It is closer to a language — a way of thinking about time, signal, and form simultaneously.

Most creative tools separate the visual from the sonic from the logical. VEKTRA treats them as one graph. A change in frequency can drive a visual parameter. A gesture can rewrite a patch. Time is explicit, not hidden. Everything is inspectable, forkable, and live.

02

How it works

01

Signal graph

Every parameter is a node. Every connection is an edge. The graph is the interface — not a metaphor for it. Audio rate, control rate, and visual rate share the same topology.

~ graph.connect osc(440) → particles.scale ✓ connected
02

Live patching

Nothing stops. You rewire while the signal flows. Nodes hot-swap. Edges reroute. The system never goes silent — it adapts. This is not "non-destructive editing." This is a living circuit.

~ midi.cc(1) → graph.rewire ✓ patch updated
03

Inspectable state

Every value is visible. Every signal can be probed. Open any node and see its waveform, its history, its connections. The system is transparent by default — because you cannot compose what you cannot see.

~ graph.probe "reverb.wet" → waveform.display
